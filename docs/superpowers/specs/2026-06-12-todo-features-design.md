# Todo機能拡張 設計書

Date: 2026-06-12

## 概要

既存のToDoアプリ（`todo/index.html`）に一般的なTodoアプリが持つ5機能を追加する。単一HTMLファイル構成を維持し、localStorage データ構造を後方互換で拡張する。

## 追加機能

| 機能 | 詳細 |
|------|------|
| インライン編集 | タスクテキストをタップ → 入力欄に変わる → Enter/blur で確定 |
| 優先度 | 追加時に🔴高/🟡中/🟢低を選択。バッジで表示。デフォルトは中 |
| 期限日 | 任意の日付ピッカー。期限切れは赤テキスト。未設定は非表示 |
| 完了済みを一括削除 | フィルターエリアに「完了を削除」ボタン。完了タスクが0件なら非表示 |
| 残タスク数 | ヘッダーに「3件残り」（未完了件数）を表示 |

## データ構造

```json
{
  "id": 1749999999000,
  "text": "買い物する",
  "done": false,
  "priority": "medium",
  "dueDate": "2026-06-15"
}
```

- `priority`: `"high"` / `"medium"` / `"low"`（既存データは`"medium"`として扱う）
- `dueDate`: ISO日付文字列 `"YYYY-MM-DD"`、または `null`

## UI変更点

**ヘッダー:** タイトルの右に「N件残り」バッジを追加。

**追加エリア（入力欄の上に展開）:**
```
[🔴 🟡 🟢]  [日付ピッカー]
[新しいタスク…         ] [+]
```

**タスク行:**
```
[☐] [🔴] タスクテキスト  [期限: 6/15]  [🗑]
```
- タスクテキストをタップするとインライン編集モードになる
- 期限切れの日付は赤で表示

**フィルターエリア:**
```
[すべて | 未完了 | 完了済み]  [完了を削除]
```
- 完了タスクが0件のとき「完了を削除」ボタンは非表示

## 後方互換

既存の `{id, text, done}` 形式のデータを読み込んだ場合:
- `priority` が未定義 → `"medium"` として扱う
- `dueDate` が未定義 → `null` として扱う
