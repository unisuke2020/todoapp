# ToDoアプリ 設計書

Date: 2026-06-12

## 概要

スマホ対応のシンプルなToDoリストWebアプリ。ビルドステップなしの単一HTMLファイル構成で、Netlifyに自動デプロイする。

## ファイル構成

```
todo/
  index.html   ← HTML + CSS + JS をすべて含む単一ファイル
```

## 機能

| 機能 | 詳細 |
|------|------|
| タスク追加 | テキスト入力 + Enterキー or +ボタン |
| 完了チェック | チェックボックスで完了/未完了を切り替え |
| タスク削除 | 各行の削除ボタン（🗑） |
| フィルター | すべて / 未完了 / 完了済み の3タブ |
| データ永続化 | `localStorage` にJSON配列で保存 |

## UI レイアウト

```
┌─────────────────┐
│  📝 ToDoリスト   │  ヘッダー
├─────────────────┤
│ [すべて|未完|完了] │  フィルタータブ
├─────────────────┤
│ ☐ 買い物する  🗑 │
│ ☑ メール返信  🗑 │  タスクリスト（スクロール可）
├─────────────────┤
│ [新しいタスク…][+]│  入力エリア（下部固定）
└─────────────────┘
```

## 技術スタック

- **HTML/CSS/JS**: 素のバニラJS、フレームワーク・ビルドツールなし
- **スタイル**: CSS変数によるダークテーマ、モバイルファースト（max-width: 480px）
- **データ**: `localStorage` キー `todo-items`、形式 `[{id, text, done}]`

## デプロイ

- **公開先**: Netlify（無料プラン）
- **自動デプロイ**: GitHubリポジトリ連携 → `git push` のたびに自動ビルド＆公開
- **初回設定**: Netlify管理画面でリポジトリ連携、`todo/` をPublish directoryに指定

## データ構造

```json
[
  { "id": 1749999999000, "text": "買い物する", "done": false },
  { "id": 1749999999001, "text": "メール返信", "done": true }
]
```

`id` はタスク追加時の `Date.now()`。
